MACHINE LEARNING
Learning
Paradigm		 Supervised
Learns from labelled training data 		Unsupervised
Discovers patterns in unlabelled data 		Reinforcement Learning
Classification Regression Clustering Dimensionality Reduction Association Rule Learning Anomaly Detection
Predicts a discrete class label, e.g., softmax regression Predicts a continuous value, e.g., linear regression Groups similar data points, e.g., k-means clustering Reduces number of features, e.g., PCA Finds co-occurrence rules, e.g., Apriori Identifies outliers, e.g., Isolation Forest Agent learns via reward signals, e.g., Q-Learning
Inference
Approach		 Instance-based Model-based
Model memorises training data and makes predictions by comparing new data points to stored instances, e.g., k-NN, SVM Model learns a general rule or function from training data that can be applied to new, unseen data, e.g., linear regression
Processing
Mode		 Batch Out-of-Core Online
Processes entire dataset at once; model trained completely, then deployed without further learning, e.g., SVM Processes large datasets in chunks that fit in memory; updates model incrementally through multiple passes, e.g., mini-batch gradient descent Updates model continuously as each new data point arrives; no need to store historical data, e.g., online gradient descent

DATA TYPES
Value Type Categorical
Represents groups or labels; numbers are just codes 		Numerical
Represents genuine amounts where arithmetic is meaningful
Ordinal Nominal Discrete Continuous
Categories where order matters, but the exact gap between them is unknown, e.g., satisfaction ratings Categories with no inherent order or ranking; purely labels, e.g., eye color or blood type Specific, countable numerical values, usually integers with clear steps, e.g., number of children Can take any value within a range, including all decimals and fractions, e.g., height or temperature
