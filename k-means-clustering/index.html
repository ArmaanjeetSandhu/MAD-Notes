Lloyd's Algorithm

Concept

The process begins by randomly initialising \(K\) cluster centroids. The algorithm then alternates between two main steps:

  1. Cluster Assignment: Each observation is assigned to the cluster whose centroid is closest, based on the smallest L2 norm (i.e., the least Euclidean distance). In other words, each observation is grouped with the centroid to which it is nearest.
  2. Centroid Update: The centroid of each cluster is updated by computing the mean of all observations within that cluster and repositioning the centroid to this mean.

These two steps are repeated iteratively until convergence, which occurs when:

  • the cluster assignments no longer change
  • the centroids remain the same as in the previous iteration
These two conditions are logically equivalent (if one holds, the other must hold too).
Variables &
Notation
\(K\) Number of clusters
\(m\) Number of observations in the dataset
\(c^{(i)}\) Index of the cluster \((1, \dots, K)\) to which observation \(\vec{x}^{(i)}\) is currently assigned
\(\vec{\mu}_k\) The vector location of cluster centroid \(k\)
\(\vec{\mu}_{c^{(i)}}\) The centroid of the cluster to which observation \(\vec{x}^{(i)}\) has been assigned
Cost Function
\(J(c^{(1)}, \dots, c^{(m)}, \vec{\mu}_1, \dots, \vec{\mu}_K) = \dfrac{1}{m}\displaystyle\sum_{i=1}^{m} \left\| \vec{x}^{(i)} - \vec{\mu}_{c^{(i)}} \right\|^2\)
Distortion / Inertia / Within-Cluster Sum of Squares (WCSS)

Because it minimises the sum of squared Euclidean distances between data points and their centroids, \(k\)-means clustering implicitly assumes clusters are spherical and of similar size. It can therefore struggle to accurately capture clusters that are elongated or of varying sizes.

Cluster Assignment
for \(i = 1 \text{ to } m : \quad c^{(i)} := \arg\min_{k} \left\| \vec{x}^{(i)} - \vec{\mu}_k \right\|^2\)
Centroid Update
for \(k = 1 \text{ to } K: \quad \vec{\mu}_k := \dfrac{1}{|C_k|} \displaystyle\sum_{i \in C_k} \vec{x}^{(i)}\)
where \(C_k\) is the set of all observations assigned to cluster \(k\), and \(|C_k|\) is the number of observations in that set.

If a cluster \(k\) has no observations assigned to it (i.e., \(C_k\) is empty), \(|C_k|\) becomes zero, leading to a division by zero in the centroid update formula. To handle this edge case, there are two approaches:

  1. Reinitialise the centroid \(\vec{\mu}_k\) to a random position within the data space, with the hope that it will attract some observations in the next iteration.
  2. Remove the empty cluster from consideration in subsequent iterations, thereby reducing the number of clusters \(K\) by one.
Choosing K

The number of clusters \(K\) is a hyperparameter that must be chosen before running the algorithm. A common heuristic for selecting \(K\) is the Elbow Method:

  1. Run the \(k\)-means algorithm for a range of values of \(K\) (e.g., \(K = 1\) to \(10\)).
  2. For each value of \(K\), compute the cost function \(J\).
  3. Plot the cost function \(J\) on the y-axis against the number of clusters \(K\) on the x-axis.

As \(K\) increases, the cost function will naturally decrease because the data points are closer to their respective centroids. The goal is to identify the "elbow" or "knee" of the curve, which is the point where the rate of decrease sharply shifts from steep to shallow. This point represents a good trade-off between minimising the cost and preventing overfitting.

However, the "elbow" is not always clearly defined. If the plot is relatively smooth without a distinct kink, other evaluation metrics like the Silhouette Score may be required to choose \(K\).

Note that \(J\) will always decrease as \(K\) increases, and so it does not make sense to pick the value of \(K\) for which \(J\) has the lowest value (i.e., \(K = m\)).

Centroid Initialisation
Random initialisation (feature space) Initialise each centroid \(\vec{\mu}_k\) to a random point within the range of the dataset's feature values.
Random initialisation (data samples) Initialise each centroid \(\vec{\mu}_k\) by randomly selecting \(K\) observations from the dataset.
Convergence

With Lloyd's algorithm, convergence is guaranteed. However, since WCSS is a non-convex function, it is possible to get stuck in a local minimum. Also, convergence is sensitive to centroid intialisation. To mitigate this, it's a good idea to run \(k\)-means multiple times with different random initialisations, and then select the clustering with the lowest cost function value.

Despite the guarantee of convergence, the number of iterations that it takes to converge can vary widely. In cases where convergence is slow, automatic convergence testing can be implemented. This involves defining a threshold \(\epsilon\) such that if \(J\) decreases by less than \(\epsilon\) between iterations, convergence is declared.
