Data Augmentation

Concept

Data augmentation is a technique used to increase the size and diversity of a dataset without actually collecting additional real-world samples. It involves generating new data points by applying transformations to existing samples that mimic realistic variations, while preserving their original labels.

  • For structured data, this could mean feature scaling, or creating new features through combinations
  • For images: rotation, flipping, cropping, zooming, noise addition, etc.
  • For text data: synonym replacement, paraphrasing, etc.

Transfer Learning

Concept

Transfer learning is a powerful technique that allows the re-purposing of a model trained on one task for a related but different task. It's especially useful when there is insufficient labelled training data for the target task, or when an existing model trained on a related task already captures rich, reusable representations of the data.

Transfer learning involves two sequential phases:

  1. In the pre-training phase, a model is trained on a large dataset for a source task. For example, training a language model on billions of words, or a vision model on millions of images. This process teaches the model general-purpose features (edges and textures for images, grammar and semantics for text) that tend to transfer well across tasks.
  2. In the fine-tuning phase, this pre-trained model is adapted to the specific target task. How this is done depends on the size of the target dataset:
    • If the dataset is quite small, the safer approach is to freeze all the pre-trained layers and only train the output layer. This way, you benefit from the rich representations already learned without risking overfitting the limited data.
    • If the dataset is larger, use the pre-trained weights as a smart starting point (rather than random initialisation), and then train all the parameters of the model. This gives the model room to adapt more deeply to the nuances of the task, while still benefiting from the head start that pre-training provides.

Multi-Task Learning (MTL)

Concept

Multi-task learning involves training a single model to perform multiple related tasks simultaneously by sharing parameters across tasks. This shared representation allows the model to leverage common patterns and improve overall performance and generalisation.

For example, for multilabel classification (in which one instance can be associated with multiple classes), instead of training separate binary classifiers for each label, a single neural network can be used. In this setup, the output layer contains one neuron per class, with a sigmoid activation applied to each neuron to produce independent probability scores for each label.
